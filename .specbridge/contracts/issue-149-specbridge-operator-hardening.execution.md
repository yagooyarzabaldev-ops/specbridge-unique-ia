## Goal

Harden the SpecBridge operator with 10 improvements: (1) ci_wait state machine — no_checks_yet is not failed_ci; (2) append-only operation ledger at .specbridge/ledger/operations.ndjson; (3) machine-readable current-goal.json generated by intake; (4) specbridge-doctor command for system health detection; (5) PR type conventions enforced by doctor; (6) execution locks at .specbridge/locks/; (7) formal idempotence status values (already_merged, already_closed, already_exists, verified_existing); (8) status dashboard HTML at docs/status-dashboard.html; (9) apply_mode and pr_types declared in policy.yaml; (10) golden path run of full 6-op loop proving all improvements work together.

## Context

The apply-mode 6-operation loop is fully operational through issue 142. Issue 143 added the intake bridge. The operator now needs hardening: ci_wait was returning failed_ci when checks had not queued yet (false negative); there was no audit trail for individual operations; state drift between CURRENT_GOAL.md and actual scope status was undetectable; no health check command existed; policy was embedded in PowerShell logic rather than declared in YAML. This batch closes all 10 gaps at once.
