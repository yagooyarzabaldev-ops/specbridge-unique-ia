# SpecBridge Operator State MCP Resource Contract

## Status

No live MCP server is implemented. This document defines the resource contract for the deterministic local CLI export surface only. A future governed contract is required before any live MCP server runtime is introduced.

## Purpose

Expose read-only SpecBridge operator state as deterministic, repository-backed MCP resource definitions for external agents.

The `specbridge-mcp-resources` CLI command emits a catalog of three resources covering current goal, doctor fix-plan health, and orchestration summaries.

## How to Use

```powershell
# Emit catalog to stdout
powershell -NoProfile -ExecutionPolicy Bypass -File ./scripts/specbridge.ps1 specbridge-mcp-resources

# Write catalog to repository artifact
powershell -NoProfile -ExecutionPolicy Bypass -File ./scripts/specbridge.ps1 specbridge-mcp-resources -OutputPath .specbridge/mcp-resources/operator-state.catalog.json
```

## Resources

### specbridge://operator/current-goal

- **Name:** `specbridge://operator/current-goal`
- **URI:** `specbridge://operator/current-goal`
- **Content type:** `application/json`
- **Source paths:** `.specbridge/state/current-goal.json`, `.specbridge/context/CURRENT_GOAL.md`
- **Refresh behavior:** on_demand
- **Sensitivity:** internal
- **Read-only:** true
- **Description:** Machine-readable current SpecBridge operator goal and task status. Includes current_task_id, run_id, title, status, primary_pr, and last_updated.

### specbridge://operator/doctor-fix-plan

- **Name:** `specbridge://operator/doctor-fix-plan`
- **URI:** `specbridge://operator/doctor-fix-plan`
- **Content type:** `application/json`
- **Source paths:** `scripts/lib/intake-doctor.ps1`, `.specbridge/scopes/*.scope.json`, `.specbridge/ledger/operations.ndjson`, `.specbridge/github-evidence/*.json`, `.specbridge/state/current-goal.json`, `docs/status-dashboard.html`
- **Refresh behavior:** on_demand; generated by `specbridge-doctor -FixPlan -Offline`
- **Sensitivity:** internal
- **Read-only:** true
- **Description:** Offline specbridge-doctor fix-plan output: detected drift classes, severity, repair commands, and safe_to_automate flags.

### specbridge://operator/orchestration-summaries

- **Name:** `specbridge://operator/orchestration-summaries`
- **URI:** `specbridge://operator/orchestration-summaries`
- **Content type:** `application/json`
- **Source paths:** `.specbridge/orchestrations/*.orchestration.json`
- **Refresh behavior:** on_demand
- **Sensitivity:** internal
- **Read-only:** true
- **Description:** Summarized view of all SpecBridge orchestration manifests: task_id, status, run_id, created_at, and source path per orchestration.

## Safety Boundaries

- The command is read-only and deterministic. It never writes to source state files.
- No live MCP server runtime is implemented.
- No network calls are made.
- No secrets, credentials, or production configuration are included.
- Writing the catalog artifact to disk requires an explicit `-OutputPath .specbridge/mcp-resources/operator-state.catalog.json` flag.
- The command does not mutate repository state when called without `-OutputPath`.

## Limitations

- Data reflects repository-backed local state at command execution time. It is not a live MCP server feed.
- Doctor fix-plan data uses offline checks only; online GitHub checks require a separate governed workflow.
- Orchestration summaries include only files with the `.orchestration.json` suffix.
